Hardware Database

Click on the Hardware Database button in the Rule Manager dialog box displays the Hardware Database dialog box. This dialog box allows adding, removing, or editing fastener(s) in the database.

 

Adding new Fasteners in the Database

1.     Open the Rule Manager dialog box.

2.     Click on the Hardware Database button to open the Hardware Database dialog box.

3.     Following are ways to add fasteners to the database:

  1. Manually type Fastener names in the Hardware Name field.

  2. Right-click on the Hardware Database dialog box and select one or more fasteners from the part location to add to the database.

  1. Add all required fasteners in an excel file. Press CTRL+I in the Hardware Database dialog box and browse the excel file. Click the Open button to import all fasteners details from the excel file to the database.

 

Note:

Following Hardware Types are supported:

    • BOLT/SCREW

    • Stud

    • Nut

    • Washer

    • WeldNut/PressNut

 

  1. Once all fasteners are added to the database, they will automatically reflect in all other fastener rules. It is a one-time activity for the rule.

  2. Click on the OK button will update the database file available at the following location:

Drive C:\ProgramData\GeometricLtd\DFMPro\DFMDataBase

When a new fastener gets added, this database file will get updated.

Input format for Fastener configuration

Wild Character support is available for fastener Configuration.

 

Input

Interpretation

 ABC

 ABC.prt or abc.prt.1 or aBc.prt.4 ñ The latest version is used gets checked.

*ABC*

 Parts contains abc or ABC in their names.

*_M32

 All parts end with _M32.

M32*

 All parts start with M32.

 

Note:

  • Mentioned Inputs are not case sensitive. Users can configure *ABC* instead of full fastener name.

  • With the above configuration, users can execute following assembly rules:

  1. Fastener Accessibility

  2. Fastener Clearance

  3. Fastener Engagement Length

  4. Fixed Fastener Calculation

  5. Floating Fastener Calculation

  6. Projected Tolerance Modifier
