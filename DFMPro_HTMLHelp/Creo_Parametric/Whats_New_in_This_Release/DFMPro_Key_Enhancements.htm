既存ルールの強化


ケーブリング

ケーブルがケーブルバンドルセグメントを使用して作成される場合、このルールはバンドルセグメント全体を 1 本のケーブルとして扱います。

対応ルール:

ケーブルと他コンポーネント間の干渉

ケーブル同士の干渉

ケーブル支持間の推奨距離


一般

ドリル貫通部の最小壁厚: このルールは、側面からの穴あけ位置までの距離を直径に対する比率としてチェックします。


ミリング

工具アクセス性: このルールは、ミリング/ドリル加工フィーチャと、ポケット/スロット近傍の壁とのクリアランスをチェックし、設計段階で干渉問題を修正できるようにします。


板金

切り欠きの最小幅: 不規則な切り欠きの場合、このルールはまず鋭角コーナー (A) のケースをチェックし、それに合格した後に切り欠きの最小幅をチェックします。


ルールマネージャーの機能強化


ルールファイル比較機能

DFMPro は、ルールマネージャーダイアログに Compare Rule Files（ルールファイル比較）ボタンを追加しました。これにより、別々の 2 つのルールファイルを比較して差分を特定できます。

詳細は Compare Rule Files を参照してください。


並べ替え順機能

DFMPro は、ルールマネージャーダイアログに Sort（並べ替え）ボタンを追加しました。これにより、列をアルファベット順に並べ替え、上矢印/下矢印を使用して優先順位を設定できます。

詳細は Sort Order を参照してください。


ヘッダー並べ替え機能

任意のヘッダー列をクリックすると矢印（上または下）が表示され、上矢印/下矢印を使用して、Rule、Category、Module、Criticality、Custom Message の各列をアルファベット順に並べ替えることができます。

詳細は Configuring Rules を参照してください。


ハードウェアデータベースの機能強化


ハードウェアタイプの一括削除機能

ユーザーは、Delete ボタン、またはキーボードの Shift と Ctrl コマンドを使用して、1 つまたは複数のハードウェアタイプを同時に削除できます。


検索オプション

DFMPro は、ハードウェアデータベースダイアログでハードウェアタイプを素早く検索するために、Search（検索）ボタンまたは Ctrl+F コマンドを提供します。


Unit 列

ハードウェアデータベースの Unit 列は、DFMPro ルール検証を効果的に行うための、異なるハードウェア単位を構成するために使用されます。

詳細は Hardware Database を参照してください。


ミリングルールに対する材料データベースサポート

DFMPro は、Narrow Regions in Pockets ルールに対する材料データベースサポートを提供しています。ルール UI では、材料リストは Map Material から取り込まれ、ユーザーは材料とその値を構成できます。


材料グループおよびグレードのサポート

DFMPro は、Materials Database から取り込まれる Material Group および Grade のドロップダウンメニューを導入しました。ユーザーは材料および関連する値を構成できます。材料グループ内では、材料グレードはアルファベット順に並び替えられます。材料グループが選択されると、DFMPro は材料グループ全体で一貫した比率を維持したまま解析を実行します。


散布図（Scatter Plot）結果表示の強化

DFMPro の結果に複数のユニットレベルの失敗インスタンスが含まれる場合、ユーザーは Scatter Plot 機能を使用して、結果をユニット単位で表示・解析し、より明確に把握できます。

詳細は Scatter Plot Result View セクションを参照してください。


DFX サーバー解析 - バージョン不一致通知

クライアントが DFX サーバーにリクエストを送信すると、サーバーはクライアント側で利用可能な DFMPro バージョンと実行サーバー上のバージョンを照合します。不一致が検出された場合、設計者に通知のポップアップメッセージが表示されます。

詳細は DFX Server Solution を参照してください。


トップレベル機能のサポート

DFMPro Settings で Execute Assembly Rules only on Top Level Assembly Parts（トップレベルアセンブリパーツのみでアセンブリルールを実行）オプションを選択すると、DFMPro はトップレベルとして識別されたサブアセンブリ内の問題は検出しません。ただし、2 つ以上のトップレベルサブアセンブリ/パーツ間の問題は検出されます。これにより、アセンブリ解析を、製品モジュール間のインターフェースレベルの問題に限定できます。この設定は、製品アセンブリが非常に大規模であり、アセンブリを完全または部分的にロードした場合にのみインターフェースレベルの問題が観察される場合に非常に有用です。

詳細は Top-Level Assembly Analysis を参照してください。
